# ManyChat Automation Playbook — @scrollmedia

## System Overview
Every Inside the Build carousel drives a ManyChat comment trigger → DM flow → guide page. Zero manual effort after initial setup and testing.

## Standard Flow Architecture
Post goes live → viewer comments [TRIGGER WORD] → auto-like comment → public comment reply → DM 1 immediate (guide link) → tag contact [vol-slug]-lead → 48hr delay if no reply → DM 2 (soft nudge to discovery call)

## Flow Naming Convention
"Inside the Build Vol.XX — [KEYWORD] Trigger"
Example: "Inside the Build Vol.01 — SIGNAL Trigger"

## Trigger Settings
Type: Instagram Comment on specific post
Post: Set to specific post AFTER it is live
Keyword: [Series trigger word]
Match: Contains keyword (NOT exact match)
Case sensitive: OFF

## Lead Tagging
Format: [slug]-lead
Examples: funnel-guide-lead, formats-library-lead, hook-system-lead
Creates segmented lists for future cross-volume broadcast messages.

## Go-Live Checklist
- Flow named correctly
- Trigger keyword matches caption and Slide 8
- Trigger set to THIS specific post only
- Auto-like ON
- Public comment reply ON
- DM 1 link confirmed live and tested on mobile
- Contact tag set after DM 1
- DM 2 has 48hr delay with no-reply condition
- Flow status: ACTIVE
- Pre-launch test from secondary account completed

## Pre-Launch Test Protocol
1. Set trigger to any old test post
2. Comment trigger word from secondary account
3. Verify: comment liked, public reply posted, DM 1 received within 60 seconds
4. Verify: guide link in DM 1 opens correctly on mobile
5. After test passes: switch trigger to actual carousel post
6. Set flow to ACTIVE

## Cross-Volume Broadcast Strategy
When Vol. 02 launches, broadcast to all funnel-guide-lead contacts.
Each new volume cross-nurtures the full accumulated lead list.
Target format: broadcast to all *-lead tags or specific volume tag.
