# @scrollmedia Content System Reference

## Account Purpose
@scrollmedia is Scroll Media's primary organic BD channel. Every post serves one of three functions:
1. Attract qualified leads (founders, SMMs, brand owners)
2. Build authority through transparent system-sharing
3. Convert engaged followers into discovery call bookings

## The Inside the Build Series — The Scroll System

Series premise: We open up how we actually work — the systems, frameworks, and processes behind every client strategy. Extreme value-add. Nothing held back. 90% of people won't execute it themselves — they'll hire us to do it.

Umbrella name: The Scroll System. Every volume is a named piece of it.

Naming rules:
- Always leads with "The Scroll"
- Second word = the specific topic (Funnel, Hook, CTA, Formats, Metrics)
- Third word = the type (System, Library, Playbook)
- Never: "The [Topic] Framework", "The [Topic] Guide", "The [Topic] Cheat Sheet"

Trigger Word Rules:
Unique ownable keyword per post. Must be: semantically connected, brand-aligned, single word, easy to type.
Approved bank: SIGNAL, MAPPED, PROOF, SEQUENCE, LAYER, COMPOUND, BLUEPRINT, SCORED, INDEXED, CALIBRATE, BENCHMARK
New words require approval before use.

## Content Mix for @scrollmedia
40% TOFU — Hook-driven, broad appeal. Cold audience.
35% MOFU — System reveals, case studies, frameworks. Warm followers.
25% BOFU — BTS client work, results, direct offers. Ready-to-hire audience.

## Post Cadence
Feed posts: 3-4x per week
Stories: 3-5 story days per week (Rachel manages)
Inside the Build carousel: 1x per series drop

## IVP A — Business Owner / Founder
Posting but getting no leads. Wants the system without being sold to.
Responds to: specific pain naming, numbered frameworks, actionable ratios.
Converts via: discovery call booking.

## IVP B — Social Media Manager / Freelancer / Agency Peer
Knows theory. Wants real agency implementation details.
Responds to: specific numbers, proprietary terminology, process depth.
Converts via: shares (referrals), collaboration DMs.

## Series Planning Process
1. Select topic — what proprietary system is useful to both IVPs?
2. Choose trigger word — passes 4-filter test (ownable, semantic, brand-aligned, low-friction)
3. Write carousel script — 8-slide structure per copywriting-standards.md
4. Build guide page — deploy to tools.scrollmedia.co/[scroll-slug], Scheme B
5. Configure ManyChat — follow manychat-playbook.md, test before posting
6. Rachel executes — script handed off as Word doc, AM owns design and posting
7. Update volume roster — add to table above
8. Cross-system update — update Claude memory, Manus Knowledge Base, GitHub skills
